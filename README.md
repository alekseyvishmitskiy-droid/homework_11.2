#Модуль masks

Базовый инструмент для скрытия конфиденциальных данных.
Маскирование карт (get_mask_card_number): Принимает 16-значный номер и возвращает его в формате XXXX XX** **** XXXX (видны первые 6 и последние 4 цифры).
Маскирование счетов (get_mask_account): Принимает номер счета и возвращает его в формате **XXXX (видны только последние 4 цифры).


#Модуль widget

Высокоуровневый обработчик входящих строк с данными карт, счетов и дат.
Умное маскирование (mask_card_and_account): Автоматически определяет тип входных данных (Карта или Счет) и применяет соответствующее правило маскирования.
Преобразование дат (get_date): Принимает дату  (например, 2024-03-11T12:00:00) и преобразует её в привычный формат 11.03.2024.


#Модуль processing

Логика для работы с массивами транзакций (списками словарей).
Фильтрация по статусу (filter_by_state): Позволяет отобрать операции со статусом EXECUTED (по умолчанию) или CANCELED.
Сортировка по дате (sort_by_date): Упорядочивает список транзакций от самых новых к самым старым.


#Модуль generators

Набор эффективных генераторов и итераторов для работы с большими массивами данных.
Фильтрация по валюте (filter_by_currency): Итератор, выдающий транзакции только в заданной валюте (например, USD, RUB).
Описание операций (transaction_descriptions): Генератор, поочередно возвращающий текстовое описание каждой транзакции.
Генератор номеров карт (card_number_generator): Создает номера карт в заданном диапазоне, форматируя их в блоки по 4 цифры (XXXX XXXX XXXX XXXX).


#Модуль decorators

Вспомогательные инструменты для мониторинга и отладки кода.
Автоматическое логирование (log): Декоратор, фиксирующий жизненный цикл функции. Записывает имя функции и результат при успехе, а при ошибке — её тип и входные параметры.
Гибкое управление выводом: Поддерживает аргумент filename. Если путь указан, логи сохраняются в файл, иначе — выводятся в консоль.
